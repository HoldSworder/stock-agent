# ETF 主线轮动策略 · 可执行模式与回测方法

> 本文档沉淀 2026-06 一轮回测打磨后**最终锁定的可执行交易模式**，是 `etf-trend-strategy-blueprint.md`（策略逻辑蓝本）的落地版：明确到每一条买卖规则、参数取值、回测方法，可直接照做、可复现。

---

## 0. 一句话定位

> **主线门禁挑对赛道 → 金叉/追涨择时进场 → Supertrend 咬住整波 → RS 更强的新主线自动轮换。**
> 系统只负责"待在当下最强的主线里"并执行纪律，不预测顶底、不追求精确抄底。

> **⚠️ 机械性边界**：本文档的回测是 **100% 机械**的——每笔买/卖/轮换都是确定性指标规则，无人工/AI 判断。它对应蓝本的"机械核心"。蓝本完整系统之上还有一层**预警层**（跌停潮/背离 → 收紧止损、减半仓），交给 AI 研判 + 人工，**未纳入本回测**。即：回测是不含人工护盘的下限版本，实盘会在其上叠加预警层。

## 1. 完整买卖规则

一笔买入必须 **同时满足"主线门禁"和"入场信号"**，缺一不可。

```
能买入 = 主线门禁通过  AND  (日线MACD金叉 OR 追涨触发)
出 场 = Supertrend(10,3) 翻空（唯一清仓信号）
轮 换 = 满仓时，出现 RS 更强的新主线候选 → 砍掉最弱持仓换上
```

### 1.1 主线门禁（决定"是不是主线、碰不碰"）
板块同时满足：
- **趋势中**：收盘价 > MA60
- **RS 守领先**：RS ≥ 自身 60 日 RS 高点 × 0.97
  （RS = 板块收盘 ÷ 沪深300收盘；"守领先"= 仍稳在近 3 个月相对强度的高位，即持续跑赢大盘）

> 门禁是慢信号、要确定性。真退潮板块 RS 早已跌破领先位，直接被挡在候选池外。
> ⚠️ 局限：`RS 守领先` 是"相对板块自身"的领先度，不是"全市场第一主线"的横向判定。横向 PK 由 §1.5 的选择层补足。

### 1.2 入场信号（决定"何时买"，二选一触发）
- **日线 MACD 金叉**：DIF 上穿 DEA（参数 12/26/9），不加任何 ADX/放量过滤
- **追涨触发**（金叉已错过时的补充入场），满足其一：
  - 斜率绿灯：20 日对数价线性回归年化斜率 ≥ 0.6 且 R² ≥ 0.55 且 乖离MA20 ≤ 15%
  - 平台突破：收盘 ≥ 20 日新高 且 昨收 < 昨日20日高 且 成交量 > 1.3×5日均量 且 乖离MA20 ≤ 15%

### 1.3 出场（唯一清仓信号）
- **Supertrend(ATR周期10, 倍数3) 由多翻空**（趋势线由绿转红）即清仓。
- 不加深死叉、不加固定止损、不用 RS 当出场闹钟（详见蓝本 §3、§4.1）。

### 1.4 主线轮换（满仓时的"换强"逻辑）
满仓且存在合格新候选时（强弱一律用 **RS动量20日** 衡量，见 §1.5 的口径说明）：
- 取持仓中 RS动量20日 最弱的一只；取候选中 RS动量20日 最强的一只
- 若 `候选 > 最弱持仓 + 0.05`（即 RS 增速强出 5 个百分点）且 该持仓已持有 ≥ 5 个交易日 → 卖最弱、买最强
- 滞后阈值（+0.05 + 持有≥5日）用于防止来回乱换

### 1.5 选择层（补空位时的横向 PK）——用 RS动量，不能用 RS 绝对值
空位补仓时，在所有"门禁通过 + 入场触发 + **RS动量20日 ≥ 0.03**"的候选里，**按 RS动量20日（= RS 的 20 日变化率，相对大盘的近期超额增速）从强到弱**挑选 → 始终待在当下增速最强的主线。RS动量≥0.03 这道门槛把"确认主线"从"守住自身高位"升级成"领先且在加速"，过滤跑输/停滞的伪主线（牛市里几乎不拦真主线，回撤不变、收益略升）。

> ⚠️ **必须用"RS 变化率"而非"RS 绝对值"做横向排名（踩过的坑）**：`RS = ETF价 ÷ 沪深300价` 这个**比值受各 ETF 名义份额单价主导**——低价 ETF（如通信ETF≈0.5元，RS≈0.12）的 RS 永远低于高价 ETF（如创业板ETF≈2.3元，RS≈0.58），与强弱无关。早期版本误用 RS 绝对值横向排名，导致**通信、半导体设备这类低价但最强的主线被系统性排除**。改用 RS 的变化率（跨标的可比）后，它们才能正常入选/换入。
> （注：§1.1 门禁里的"RS守领先"是 RS 跟**自身历史**比，自相对、不受单价影响，仍正确。）
>
> **20日 vs 60日 的选择**：回测中 **RS动量20日 在多仓配置上更优**，灵敏、抓轮动快，代价是换手更高、出样本可能更脆。**RS超额60日**（RS 的 60 日变化率）更平滑、换手更少。默认用 **RS动量20日**。

### 1.6 仓位（不强推单一档，按可承受回撤自选）
- 持有 **N 只**（N=1/2/3），**等权**，每只 1/N 仓位。
- 出场或被轮换后资金回到现金，等待下一个合格信号补位；无合格候选时该格留现金（可阶段性空仓）。
- 收益与回撤近似线性绑定，**抓真主线 ⇄ 大回撤焊死**，按可承受回撤选档（完整跟踪池实测，见 §3）：
  - **进攻**：满仓2只 +190%/−24% 或 满仓3只 +178%/−22%（3只更分散、回撤略低）
  - **均衡**：3只各28%留16% +145%/−19% ｜ 3只各25%留25% +125%/−18%
  - **稳健**：3只各22%留34% +106%/−16%
  - 满仓1只 +172%/−35%：收益不差但回撤陡、最颠，不推荐。

## 2. 回测方法（可复现）

| 项 | 设定 |
|---|---|
| **数据源** | a-stock-data sidecar `mootdx_kline`（`http://<NAS>:9119`，通达信本地 TCP，不封 IP，~0.3s/只，比 NAS `/api/kline` 快 ~20×；为同一份 mootdx 数据，已验证两源逐日一致） |
| **份额折算复权** | mootdx 原始数据对 ETF 份额折算**不复权**，折算日会出现假 −50% 断崖（创业板AI/金融科技/港股创新药银华等）。回测取数时检测"单日跌幅>35%即折算"并按比例前复权成连续序列。**⚠️ NAS `/api/kline` 同样未复权，生产盯盘/指标在折算标的上也受影响，待修。** |
| **区间** | 2025-01-02 → 2026-06-26（**连续一段**，让盈利跨年复利，不拆年） |
| **基准/RS分母** | 沪深300（510300） |
| **标的池** | **ETF 模块完整跟踪池（`/api/etf/pool`，55 只）**（见 §5） |
| **入场门槛** | 候选除过"主线门禁"外，还要 **RS动量20日 ≥ 0.03**（相对大盘近期超额增速明显为正＝主线在加速，过滤"守住高位但已停滞"的伪主线） |
| **成交价** | 默认 **当日收盘价**（T 日信号 T 日收盘成交，含轻微理想化：需等收盘才知信号，却以收盘价成交）。实盘对应**尾盘 14:50 信号定型后挂收盘价附近**。 |
| **账本** | 初始 1.0；等权 N 格各 1/N×当前权益；现金不计息；单格买入额 = min(权益×1/N, 可用现金) |
| **回撤** | 按每日权益曲线峰值计最大回撤 |
| **年化** | (期末净值)^(1/年数) − 1，年数 = 交易日数 / 244 |

### 2.1 指标口径
- **MACD**：EMA12 − EMA26 = DIF；DEA = EMA9(DIF)；金叉 = DIF 上穿 DEA。
- **Supertrend(10,3)**：ATR = SMA(TR,10)；上下轨 = (H+L)/2 ± 3×ATR；标准翻转规则取方向（+1多 / −1空）。
- **RS**：板块收盘 ÷ 沪深300收盘。
- **RS守领先**（门禁，自相对）：RS ≥ max(近60日RS) × 0.97。
- **RS动量20日**（横向排名/轮换 + 入场门槛，**默认**，跨标的可比）：RS_今 ÷ RS_20日前 − 1；入场要求 ≥ 0.03。
- **RS超额60日**（横向排名/轮换，备选·更平滑）：RS_今 ÷ RS_60日前 − 1。
- **份额折算前复权**：扫描日线收盘，单日比值 <0.65 或 >1.5 判为折算日，将折算前 OHLC 整体按比例缩放成连续序列（量同步反向缩放）。
- **斜率/R²**：对最近 20 日 log 收盘做最小二乘线性回归，斜率年化（×252）。

### 2.2 执行顺序（每个交易日盘后）
1. 先处理出场：持仓中任意一只 ST 翻空 → 清仓该格。
2. 计算候选池：未持有 + 门禁通过 + 入场触发 + RS动量20日≥0.03，按 RS动量20日 降序。
3. 主线轮换：满仓且候选满足换强条件（候选 > 最弱持仓 + 0.05 且持有≥5日）→ 砍最弱换最强。
4. 补空位：有空格则按 RS动量20日 最强依次补满（无合格候选则该格留现金，可阶段性空仓）。
5. 记录当日权益、更新最大回撤。

## 6. 局限与诚实提醒

1. **样本是两个超级趋势年**：2025/2026 是 A 股罕见的连续主升行情，+100% 量级年化是"强行情 + 强赛道池"叠出来的，**不是常态**；换大熊市或风格剧变会大幅打折。
2. **浮盈占比**：满仓档总收益里有相当部分是期末未实现浮盈（半导体设备 +27%、通信 等仍持有中），若回吐总收益会下来。
3. **抓真主线 ⇄ 大回撤焊死**：满仓档真实回撤 ~22~24%；压到 ~18% 收益降到 +125%（留25%现金）。收益与回撤近似线性。
4. **份额折算复权（已修，但生产待修）**：mootdx 原始数据对 ETF 份额折算不复权，折算日有假 −50% 断崖（创业板AI/金融科技/港股创新药银华等），会把回测污染到单笔 −49%。回测已加"单日>35%即折算前复权"修掉。**⚠️ NAS `/api/kline`（生产盯盘/指标取数）同样未复权，这些折算标的在折算日会被误判暴跌，建议在生产侧补同样的折算复权。**
5. **横向指标曾踩"份额单价"坑**：RS 绝对值不可跨标的比较，必须用 RS 变化率（默认 RS动量20日，门槛≥0.03，见 §1.5）。这是被用户实盘直觉（"最强的通信/半导体设备怎么没在记录里"）反查出来的。
6. **同族板块相关性风险**（待优化，跟踪池尤甚）：横向排名可能同时选入光伏/电池/储能、或两格电网等高相关品种，被同一波回调一起 ST 放大回撤（见 §4 的 2025-11、2026-03）。跟踪池重复标的多会加重，建议清理重复 + 后续加"同族去重/相关性约束"。
7. **主线判定是相对自身**：`RS守领先`（门禁）是 RS 跟自身比，不等于"全市场第一主线"；横向排名 + RS动量≥0.03 门槛（§1.5）已缓解，但只在当天触发的候选间比。
8. **多腿趋势捕获打折**（结构性）：锯齿磨涨的板块（如电池）会因 ST 被反复洗，吃不满整波——回撤保护与捕获率不可兼得。
9. **数据源**：默认 a-stock-data sidecar `mootdx_kline`（不封 IP、快），与 NAS `/api/kline` 同源一致；需关注 sidecar 与国内网络稳定性。

## 7. 一页速查

```
数据源       → a-stock-data mootdx_kline（不封IP/快/同NAS一致）+ 份额折算前复权
门禁(碰不碰) → 收盘>MA60 且 RS≥自身60日高×0.97（主线，否则不碰）
进场(何时买) → (日线MACD金叉 或 追涨) 且 RS动量20日≥0.03（领先且在加速）
选择(买哪只) → 合格候选里 RS动量20日 最强的补空位（不能用RS绝对值!）
出场(何时卖) → Supertrend(10,3) 翻空（唯一清仓）
轮换(换不换) → 满仓时 候选RS动量20日 > 最弱持仓 + 0.05 且持有≥5日 → 砍弱换强
仓位         → 等权N只(无候选则空仓)；满仓2只+190/−24 满仓3只+178/−22 3只留25%现金+125/−18
池           → ETF模块完整跟踪池(55)；自行清理重复标的可降换手与同族回撤
```
